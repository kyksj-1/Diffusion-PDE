# 多环境开发：从第一天就做对

> 适用于所有 ML/科研/量化项目。基于 EOF 项目的踩坑经验，提炼出通用原则。
> 文末附两个新项目（扩散模型解 PDE、量化金融）的具体模板。

---

## 第一部分：环境迁移到底容易踩哪些坑

### 1. 路径问题（占所有迁移故障的 80%）

这是**最高频、最致命、最容易被忽视**的问题。

**坑 1：绝对路径写进代码**

```python
# 你在自己电脑上写的，跑得好好的
data = load("D:/my_project/data/train.hdf5")       # Windows
data = load("/home/zhangsan/data/train.hdf5")       # 服务器
data = load("/mnt/nas-new/home/liuyanzhi/data/...") # EOF项目的真实案例
```

换一台机器，100% 报 FileNotFoundError。这不是"以后再改"的问题——**一旦绝对路径散布到 10 个文件里，重构的痛苦远大于一开始就做对**。

**坑 2：相对路径依赖于"当前工作目录"**

```python
data = load("./data/train.hdf5")  # 看起来很合理？
```

问题：`./` 指的是你**运行命令时所在的目录**，不是脚本所在的目录。

- 在 `ONFM/` 下运行 `python train.py` → `./data/` 指向 `ONFM/data/`
- 在项目根目录运行 `python ONFM/train.py` → `./data/` 指向根目录的 `data/`
- 在 Colab 中 `%cd /content` 后运行 → `./data/` 指向 `/content/data/`

三个地方跑出三个不同的路径，且**不报语法错误，只是找不到文件**，排查起来非常浪费时间。

**坑 3：Windows 与 Linux 的路径分隔符**

```python
path = "data\\train\\file.hdf5"  # Windows 写法
path = "data/train/file.hdf5"    # Linux 写法
```

在 Windows 上用 `\` 拼出来的路径，传到 Linux 上直接报错。反过来 `/` 在 Windows 上倒是大部分时候能用，但不是所有库都兼容。

**坑 4：数据和代码耦合在一起**

```
project/
├── src/
├── data/          ← 几十GB的数据集放在代码仓库里
├── checkpoints/   ← 几GB的模型权重
└── .git/          ← Git 仓库膨胀到无法 clone
```

一旦不小心 `git add .`，大文件进了 Git 历史，即使之后删除，`.git` 目录依然巨大。GitHub 免费仓库有 1GB 限制，超了就推不上去。

---

### 2. 依赖问题

**坑 5：torch 版本和 CUDA 版本不匹配**

```
PC：      CUDA 12.1 + torch 2.6.0+cu121
Colab：   CUDA 12.2 + torch 2.5.1+cu122（Colab 预装的）
服务器：  CUDA 11.8 + torch 2.1.0+cu118（老驱动）
```

`pip install torch` 安装的版本取决于当前系统的 CUDA。如果你在 PC 上 freeze 了依赖然后在服务器上 `pip install -r requirements.txt`，很可能装上一个 CUDA 版本不匹配的 torch，结果 GPU 不可用但不报错，默默回退到 CPU 跑。

**坑 6：Colab/Kaggle 的预装包与你的版本冲突**

Colab 预装了 torch、numpy、pandas 等。如果你的 `requirements.txt` 指定了不同版本，`pip install` 可能把 Colab 的环境搞乱。

**坑 7：云服务器没有外网**

`pip install` 直接超时。`git clone` 也不行。这在国内高校的计算集群中**极其常见**。

---

### 3. GPU 与硬件问题

**坑 8：显存不够但代码没有优雅降级**

PC 上 8GB 显存，batch_size=40 直接 OOM。代码里 batch_size 写死了，改起来要翻好几个文件。

**坑 9：多人共享 GPU 互相挤占**

你在服务器上跑实验占了 GPU 0，同学也在 GPU 0 上跑，两个人都 OOM。没有一个地方协调谁用哪张卡。

**坑 10：`num_workers` 在 Windows 上报错**

```
RuntimeError: 
    An attempt has been made to start a new process before the
    current process has finished its bootstrapping phase.
```

Windows 上 `num_workers > 0` 必须在 `if __name__ == '__main__':` 保护下运行。很多科研代码是在 Linux 上写的，根本不考虑这一点。

---

### 4. 数据同步问题

**坑 11：数据集在不同机器上版本不一致**

你在 PC 上处理了数据（裁剪、降采样），忘了同步到服务器，服务器上跑的还是旧版数据。训练结果和 PC 上对不上，花一整天排查。

**坑 12：大文件传输中断后从头开始**

`scp` 传一个 20GB 文件，传到 18GB 断网了。`scp` 不支持断点续传，只能重来。

---

### 5. 结果可复现性问题

**坑 13：同样的代码在不同环境跑出不同结果**

- 不同的 cuDNN 版本 → 卷积运算的数值精度不同
- 不同的 `num_workers` → DataLoader 的数据顺序可能不同
- 没有固定随机种子 → 每次跑结果都不一样

### 3. env_manager.py 完整实现思路

这个文件是整个方案的枢纽。所有其他代码只跟它打交道，不直接读 YAML、不直接拼路径。

```python
"""
env_manager.py — 项目环境管理器

使用方式（在任何文件中）：
    from src.utils.env_manager import env
    
    data_path = env.get_dataset("pde_heat_2d", "train")  # 自动拼接绝对路径
    log_dir   = env.output_root / "logs"                  # pathlib.Path 对象
    device    = env.device                                 # torch.device
    
设计原则：
    1. 全局单例，import 即可用
    2. 所有路径返回 pathlib.Path（跨平台兼容）
    3. 找不到配置文件时 fallback 到合理默认值
    4. 支持 {user} 等变量插值
"""

import os
import yaml
import getpass
from pathlib import Path

class EnvManager:
    def __init__(self):
        # 查找 env_config.yaml 的顺序：
        # 1. 环境变量 ENV_CONFIG_PATH 指定的路径
        # 2. 项目根目录/configs/env/env_config.yaml
        # 3. 如果都没有 → 使用安全的默认值
        config_path = self._find_config()
        
        if config_path and config_path.exists():
            with open(config_path, 'r', encoding='utf-8') as f:
                self._raw = yaml.safe_load(f)
        else:
            print("[EnvManager] 未找到 env_config.yaml，使用默认配置")
            self._raw = {'active_env': 'pc', 'environments': {}, 'defaults': {}}
        
        self._active_name = self._raw.get('active_env', 'pc')
        self._defaults = self._raw.get('defaults', {})
        self._env = self._raw.get('environments', {}).get(self._active_name, {})
        self._datasets = self._raw.get('datasets', {})
    
    # ---------- 路径相关 ----------
    
    @property
    def data_root(self) -> Path:
        return Path(self._resolve(self._env.get('data_root', './data')))
    
    @property
    def output_root(self) -> Path:
        p = Path(self._resolve(self._env.get('output_root', './output')))
        p.mkdir(parents=True, exist_ok=True)
        return p
    
    def get_dataset(self, name: str, split: str = 'train') -> Path:
        """获取数据集的绝对路径。找不到就报清晰的错误。"""
        if name not in self._datasets:
            available = list(self._datasets.keys())
            raise KeyError(f"数据集 '{name}' 未在 env_config.yaml 中注册。可用: {available}")
        
        ds = self._datasets[name]
        if split not in ds and 'source' in ds:
            rel = ds['source']  # 不分 train/test 的数据集
        elif split in ds:
            rel = ds[split]
        else:
            raise KeyError(f"数据集 '{name}' 没有 '{split}' 分割")
        
        full_path = self.data_root / rel
        if not full_path.exists():
            print(f"[警告] 数据路径不存在: {full_path}")
        return full_path
    
    # ---------- 硬件相关 ----------
    
    @property
    def gpu_ids(self) -> str:
        return str(self._env.get('gpu_ids', '0'))
    
    @property
    def device(self):
        import torch
        os.environ["CUDA_VISIBLE_DEVICES"] = self.gpu_ids
        return torch.device('cuda' if torch.cuda.is_available() else 'cpu')
    
    @property
    def num_workers(self) -> int:
        return int(self._env.get('num_workers', 0))
    
    @property
    def batch_size_scale(self) -> float:
        return float(self._env.get('batch_size_scale', 1.0))
    
    @property
    def precision(self) -> str:
        return self._env.get('precision', self._defaults.get('precision', 'fp32'))
    
    # ---------- 工具方法 ----------
    
    def _resolve(self, s: str) -> str:
        """变量插值：{user} → 当前用户名"""
        return s.replace('{user}', getpass.getuser())
    
    def _find_config(self) -> Path:
        """按优先级查找配置文件"""
        # 优先级1：环境变量
        if 'ENV_CONFIG_PATH' in os.environ:
            return Path(os.environ['ENV_CONFIG_PATH'])
        # 优先级2：从当前文件位置向上查找
        current = Path(__file__).resolve().parent
        for _ in range(5):  # 最多向上找5层
            candidate = current / 'configs' / 'env' / 'env_config.yaml'
            if candidate.exists():
                return candidate
            current = current.parent
        return None
    
    def summary(self):
        """打印当前环境摘要，用于训练开始前确认"""
        print(f"{'='*50}")
        print(f"  环境: {self._active_name}")
        print(f"  数据: {self.data_root}")
        print(f"  输出: {self.output_root}")
        print(f"  GPU:  {self.gpu_ids}")
        print(f"  Workers: {self.num_workers}")
        print(f"  精度: {self.precision}")
        print(f"{'='*50}")

# 全局单例 —— import 后直接用
env = EnvManager()
```

```bash
#!/bin/bash
# 根据当前环境自动选择正确的 torch 版本

# 检测 CUDA 版本
if command -v nvcc &> /dev/null; then
    CUDA_VERSION=$(nvcc --version | grep -oP 'release \K[0-9]+\.[0-9]+')
    echo "检测到 CUDA $CUDA_VERSION"
else
    CUDA_VERSION="none"
    echo "未检测到 CUDA，将安装 CPU 版本"
fi

# 安装 torch（根据 CUDA 版本）
case $CUDA_VERSION in
    12.*)
        pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cu121
        ;;
    11.8*)
        pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cu118
        ;;
    *)
        pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cpu
        ;;
esac

# 安装其他依赖
pip install -r requirements.txt

echo "安装完成！运行 python -c \"import torch; print(torch.cuda.is_available())\" 验证GPU"
```

### 6. 环境自检脚本

新到一台机器，跑一下就知道缺什么：

```python
# tests/test_env.py
"""
运行: python tests/test_env.py
一键检查当前环境是否满足项目运行条件
"""

def check_all():
    results = []
    
    # 1. Python 版本
    import sys
    ok = sys.version_info >= (3, 9)
    results.append(("Python >= 3.9", ok, sys.version.split()[0]))
    
    # 2. torch + CUDA
    try:
        import torch
        cuda_ok = torch.cuda.is_available()
        gpu_name = torch.cuda.get_device_name(0) if cuda_ok else "N/A"
        vram = f"{torch.cuda.get_device_properties(0).total_mem / 1e9:.1f}GB" if cuda_ok else "N/A"
        results.append(("PyTorch", True, torch.__version__))
        results.append(("CUDA 可用", cuda_ok, f"{gpu_name} ({vram})" if cuda_ok else "不可用"))
    except ImportError:
        results.append(("PyTorch", False, "未安装"))
    
    # 3. 核心依赖
    for pkg in ['yaml', 'numpy', 'h5py', 'einops', 'scipy', 'pandas']:
        try:
            mod = __import__(pkg)
            ver = getattr(mod, '__version__', '?')
            results.append((pkg, True, ver))
        except ImportError:
            results.append((pkg, False, "未安装"))
    
    # 4. env_config.yaml
    from src.utils.env_manager import env
    config_exists = env._find_config() is not None
    results.append(("env_config.yaml", config_exists, str(env._active_name) if config_exists else "未找到"))
    
    # 5. 数据目录
    data_ok = env.data_root.exists()
    results.append(("数据目录存在", data_ok, str(env.data_root)))
    
    # 打印报告
    print("\n" + "="*60)
    print("  环境自检报告")
    print("="*60)
    for name, ok, detail in results:
        status = "OK" if ok else "FAIL"
        print(f"  [{status:>4}] {name:<20} {detail}")
    print("="*60)
    
    fails = [r for r in results if not r[1]]
    if fails:
        print(f"\n  有 {len(fails)} 项未通过，请修复后再运行项目。")
    else:
        print(f"\n  全部通过！可以开始训练。")

if __name__ == '__main__':
    check_all()
```

## 第三部分：通过 YAML 一键调配——实验配置

除了环境配置，**实验参数**也应该通过 YAML 管理。这是两层分离：

```
env_config.yaml      → 回答"在哪里跑"（路径、GPU、线程数）
experiment.yaml      → 回答"怎么跑"（模型参数、学习率、epoch）
```

### 实验配置加载机制

```python
# src/utils/config_loader.py

import yaml
from pathlib import Path
from src.utils.env_manager import env

def load_experiment(config_name: str) -> dict:
    """
    加载实验配置，并自动注入环境信息。
    
    用法: config = load_experiment("diffusion_pde_base")
    """
    config_path = Path(__file__).resolve().parents[2] / "configs" / "experiment" / f"{config_name}.yaml"
    
    with open(config_path, 'r', encoding='utf-8') as f:
        config = yaml.safe_load(f)
    
    # 自动注入环境参数
    config['device'] = str(env.device)
    config['num_workers'] = env.num_workers
    config['output_dir'] = str(env.output_root / config.get('name', 'unnamed'))
    
    # batch_size 自动缩放
    if 'batch_size' in config:
        config['batch_size'] = max(1, int(config['batch_size'] * env.batch_size_scale))
    
    return config
```

```yaml
  attention_resolutions: [16, 8]
  num_res_blocks: 2

  noise_schedule: cosine        # 噪声调度策略
  # PDE 特有
```

**这个项目环境迁移的特殊注意事项**：

1. **扩散模型训练非常吃显存**。1000 步扩散 + UNet → PC 上需要把 `batch_size_scale` 设为 0.125 甚至更低，或者减少 `diffusion_steps`。建议在 PC 上用 `diffusion_steps: 100` 的简化版做调试。

2. **高维 PDE 数据集可能非常大**。如果是 3D Navier-Stokes（128x128x128），单个样本就上百 MB。PC 上建议只放 2D 数据做调试，3D 实验放到服务器上跑。

3. **可以在 `env_config.yaml` 中为不同环境指定不同的默认数据集**：
   ```yaml
   pc:
     default_dataset_override:
       pde_navier_stokes: pde_navier_stokes_2d_small  # PC上用小数据集
   server:
     # 不覆盖，用完整数据集
   ```

**这个项目环境迁移的特殊注意事项**：

1. **数据获取是最大挑战**。股票数据通常从 tushare / akshare / wind 等接口获取，这些接口**需要外网**。


3. **Colab / Kaggle 不太适合量化项目**。回测需要大量 I/O 操作，云平台磁盘速度慢。建议量化项目主要在 PC 和服务器上跑。

4. **结果保密性**。量化策略有商业价值，确保：
   - 训练好的模型权重**不要**推到公共 Git 仓库
   - 因子计算逻辑如果涉及 alpha 信号，考虑单独加密或不入库

## 第六部分：总结——五条铁律

1. **路径永远不硬编码**。所有路径都从 `env_manager` 获取，用 `pathlib.Path` 处理跨平台兼容。

2. **数据和代码彻底分离**。代码在 Git 里，数据在 `data_root` 里，两者通过 `env_config.yaml` 连接。

3. **环境差异收敛到一个文件**。GPU 编号、线程数、batch_size、精度——所有因机器而异的参数，全部收进 `env_config.yaml`。

4. **第一天就写 .gitignore**。数据文件、模型权重、日志、`env_config.yaml` 一律不入库。

5. **每台机器到手先跑自检脚本**。在发现缺依赖/路径错误之前，不要开始正式实验。

这五条如果从项目第一天就遵守，以后换任何环境都是改一行 `active_env` 的事。
